¿Cuál es la diferencia entre Internet y la World Wide Web?

Internet es una red global de computadoras interconectadas que permite la comunicación y el intercambio de datos entre ellas. La World Wide Web, o simplemente la web, es un sistema de información donde los documentos y otros recursos web son identificados por URLs, interconectados por enlaces y accesibles a través de Internet.

¿Cuáles son las partes de un URL?

Un URL incluye el protocolo (como http o https), el nombre de dominio (o dirección IP), y opcionalmente, el puerto, el path al recurso, una cadena de consulta (query string) y un fragmento.

¿Cuál es el propósito de los métodos HTTP: GET, HEAD, POST, PUT, PATCH, DELETE?

GET solicita la representación de un recurso. HEAD pide los encabezados de respuesta similares a los de GET, pero sin el cuerpo de respuesta. POST envía datos al servidor para crear o actualizar un recurso. PUT reemplaza todas las representaciones actuales del recurso de destino con la carga útil de la petición. PATCH aplica modificaciones parciales a un recurso. DELETE elimina un recurso especificado.

¿Qué método HTTP se debe utilizar al enviar un formulario HTML, por ejemplo cuando ingresas tu usuario y contraseña en algún sitio? ¿Por qué?

Generalmente se utiliza POST para enviar un formulario que incluye datos sensibles como usuario y contraseña, ya que POST no incluye la información enviada en la URL y permite el envío de datos más grandes de forma segura.

¿Qué método HTTP se utiliza cuando a través de un navegador web se accede a una página a través de un URL?

El método GET se utiliza cuando se accede a una página a través de un URL en un navegador web, solicitando la representación de un recurso específico.

Un servidor web devuelve una respuesta HTTP con código 200. ¿Qué significa esto? ¿Ocurrió algún error?

Un código 200 significa que la solicitud ha tenido éxito. No ocurrió ningún error.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 404? ¿Por qué?

Sí, es responsabilidad del desarrollador, ya que un error 404 indica que un recurso no fue encontrado en el servidor, lo cual podría deberse a un enlace roto o un error en la dirección URL que necesita ser corregido.

¿Es responsabilidad del desarrollador corregir un sitio web si un usuario reporta que intentó acceder al sitio y se encontró con un error 500? ¿Por qué?

Sí, porque un error 500 indica un error interno del servidor, lo que sugiere un problema en la configuración del servidor o en el código del sitio web que el desarrollador necesita investigar y solucionar.

¿Qué significa que un atributo HTML5 esté depreciado o desaprobado (deprecated)? Menciona algunos elementos de HTML 4 que en HTML5 estén desaprobados.

Significa que el atributo o elemento ya no se recomienda para su uso y podría dejar de ser funcional en futuras versiones de navegadores. Ejemplos incluyen la etiqueta <font>, los atributos como align en varias etiquetas, y la etiqueta <center>.

¿Cuáles son las diferencias principales entre HTML 4 y HTML5?

HTML5 introduce nuevas etiquetas semánticas, mejora el soporte para aplicaciones web con APIs, incluye soporte nativo para multimedia con <video> y <audio>, y elimina etiquetas y atributos obsoletos.

¿Qué componentes de estructura y estilo tiene una tabla?

Una tabla se compone de la etiqueta <table>, con filas (<tr>), columnas (<td> o <th> para celdas de encabezado), opcionalmente agrupadas en <thead>, <tbody>, y <tfoot>.

¿Cuáles son los principales controles de una forma HTML5?

Los controles incluyen <input> (con diversos tipos como text, password, email, etc.), <textarea>, <select>, <button>, entre otros. HTML5 añade nuevos tipos de input como date, number, range, y email.

Sobre el ciclo de vida y desarrollo de los sistemas de información:

El ciclo de vida de los sistemas de información incluye la planificación, análisis, diseño, implementación, y mantenimiento. El ciclo de desarrollo de sistemas de información sigue fases similares, enfocándose en el desarrollo práctico del sistema, desde su concepción hasta su despliegue y revisión continua.

Si se pone una imagen de fondo en una página HTML, ¿por qué debe escogerse con cuidado?

Para que no afecte la legibilidad del texto, que se vea bien en diferentes tamaños de pantalla y que no incremente innecesariamente los tiempos de carga de la página

% para diseños responsivos y fluidos

px para tamaños fijos y precisión en elementos que no requieren ajustarse con el tamaño de pantalla

Evitar pt que es para impresión

¿Por qué el uso de una versión minimizada del CSS mejora el rendimiento del sitio?

Reduciendo el tamaño de los CSS disminuye el tiempo de carga de la página al reducir el ancho de banda necesario para descargar los estilos

Teorema de Pitágoras

El teorema de Pitágoras es una relación en geometría euclidiana entre los tres lados de un triángulo rectángulo. Afirma que el área del cuadrado cuyo lado es la hipotenusa es igual a la suma de las áreas de los cuadrados cuyos lados son los catetos.
